# 开发者交接文档

## 1. 技术栈

| 层 | 技术 | 说明 |
|---|------|------|
| 前端 | React 19 + TypeScript + Vite 6 + Tailwind 4 | SPA，shadcn/ui 组件库 |
| 后端 | Python 3 + FastAPI + SQLAlchemy 2 | REST API，JWT 认证 |
| 数据库 | SQLite（每用户独立文件） | 系统库：`recruitment_system.db`，用户库：`recruitment_{用户名}.db` |
| AI 解析 | 硅基流动 SiliconFlow API | Qwen3-8B（文本）/ Qwen3-VL-8B（图片PDF） |
| 部署 | 阿里云 ECS + Nginx + uvicorn | 前端静态文件由 Nginx 服务，API 代理到 uvicorn |

## 4. 部署命令

```bash
# 本地构建
cd frontend && npx vite build

# 打包
tar -czf deploy.tar.gz --exclude='__pycache__' --exclude='*.pyc' --exclude='data' backend/ frontend/dist/

# 上传
scp deploy.tar.gz root@8.149.143.126:/tmp/

# 服务器部署
ssh root@8.149.143.126
cd /var/www/v2.0-lite
rm -rf frontend/dist backend_old
cp -r backend backend_old  # 备份
tar -xzf /tmp/deploy.tar.gz

# 重启后端
kill $(cat backend/uvicorn.pid 2>/dev/null) 2>/dev/null
cd backend
nohup python3 -m uvicorn app.main:app --host 0.0.0.0 --port 8000 </dev/null > /var/log/recruitment.log 2>&1 &
echo $! > uvicorn.pid
disown
```

## 5. 关键架构决策

### 每用户独立数据库
每个用户在 `backend/data/` 下有独立 SQLite 文件。**优点**：数据物理隔离，不同用户不会互相影响。**问题**：自增 ID 在不同库间重复，聚合时必须带 `intern_name` 定位。

### 日期格式铁律
- **存储/API**：永远 `YYYY-MM-DD`（横杠）
- **前端显示**：`YYYY/MM/DD`（斜杠，仅显示层转换）
- **违反者**：统计全为 0、日期筛选失效、API 崩溃

### 多用户缓存
- localStorage key 必须含用户名
- 模块级缓存（`_cachedReports`）必须校验 `_cachedForUser`
- `useEffect` 每帧检测用户变化，立即清空旧缓存

## 6. 已知陷阱 & Bug 速查

| 陷阱 | 症状 | 原因 | 修复要点 |
|------|------|------|---------|
| 日期斜杠污染 | 统计全为 0 | DB 存 `2026/05/26`，后端比较 `>= 2026-05-25` 失败 | `todayStr()` 必须输出 YYYY-MM-DD |
| 缓存串号 | A 看到 B 的数据 | LS key 不含用户名 + 模块缓存未校验归属 | `getLSKey()` 含用户名 + useEffect 检测用户切换 |
| 跨库 ID 重复 | 删一个全没 | `filter(c => c.id !== 1)` 删掉所有库的 id=1 | 后端用 intern_name 定位库，前端用复合键过滤 |
| API 参数不匹配 | 筛选无效 | 前后端参数名不一致 | `name`→`owner_id`, `date_start`→`date_from` |
| HTTP 复制无效 | 复制按钮没反应 | `navigator.clipboard` 需要 HTTPS | 用 `execCommand('copy')` |
| Nginx 413 | 大 PDF 上传失败 | 默认 1MB 限制 | `client_max_body_size 20m` |
| 复试通过率错误 | 率不对 | 分母用错字段 | 分母是 `second_interview_show` |
